useState can hold any JavaScript value

String state

Hello, ...

const [name, setName] = useState('');

Boolean state

false
const [isOn, setIsOn] = useState(false);
setIsOn(prev => !prev);

Array state

Always create a NEW array — never mutate the existing one.
setTags([...tags, newTag]); // ✅ new array

Object state

setUser({...user, age: user.age+1});
// ❌ This runs expensiveCalc() on EVERY render const [data, setData] = useState(expensiveCalc()); // ✅ Pass a function — React calls it only on the FIRST render const [data, setData] = useState(() => expensiveCalc());
The function form is called lazy initializer. React calls it once, then ignores it on re-renders.

Pitfall 1 — Calling hooks conditionally

// ❌ Hooks must be called in the same order every render if (isLoggedIn) { const [name, setName] = useState(''); // breaks the rules! } // ✅ Declare unconditionally, use conditionally const [name, setName] = useState(''); if (isLoggedIn) { /* use name here */ }

Pitfall 2 — Reading state right after setting it

setCount(count + 1); // count is still the OLD value here! console.log(count); // ❌ stale
const next = count + 1; setCount(next); console.log(next); // ✅ correct
State updates are async — the new value only arrives on the next render.

Pitfall 3 — Mutating state directly

// ❌ Mutating — React won't detect the change, no re-render! user.name = 'Alice'; setUser(user); // same reference → React skips render // ✅ Create a new object/array setUser({ ...user, name: 'Alice' }); // new reference → triggers render

Pitfall 4 — Too many separate state variables

// ❌ Too granular const [firstName, setFirstName] = useState(''); const [lastName, setLastName] = useState(''); const [email, setEmail] = useState(''); // ✅ Group related state const [form, setForm] = useState({ firstName: '', lastName: '', email: '' });
