Subscription pattern

useEffect(() => { const handler = (e) => setPos({ x: e.clientX, y: e.clientY }); window.addEventListener('mousemove', handler); return () => { window.removeEventListener('mousemove', handler); }; }, []);
Without cleanup, every re-mount adds another listener. After 10 remounts: 10 listeners firing simultaneously.

Pattern 1 — data fetching with race guard

Ready — click a button to fetch
useEffect(() => { let cancelled = false; fetchUser(userId) .then(data => { if (!cancelled) setUser(data); }); return () => { cancelled = true; }; }, [userId]);
Always guard stale requests — if userId changes fast, the old fetch might resolve after the new one.

Pattern 2 — document title sync

document.title = "useEffect Playground"
useEffect(() => { document.title = title; }, [title]);

Pattern 3 — debounce (500ms)

Waiting for input...
useEffect(() => { const t = setTimeout(() => searchAPI(query), 500); return () => clearTimeout(t); }, [query]);

Pitfall 1 — infinite loop

// ❌ Infinite loop! const [data, setData] = useState([]); useEffect(() => { setData([1, 2, 3]); }); // no array!
// ✅ Fixed — runs once const [data, setData] = useState([]); useEffect(() => { setData([1, 2, 3]); }, []); // ✓
Missing the dep array: effect sets state → re-render → effect again → forever.

Pitfall 2 — stale closure

// ❌ count always reads 0 useEffect(() => { const id = setInterval(() => { console.log(count); // stale! }, 1000); return () => clearInterval(id); }, []); // count=0 frozen here
// ✅ Use ref for latest value const countRef = useRef(count); countRef.current = count; useEffect(() => { const id = setInterval(() => { console.log(countRef.current); }, 1000); return () => clearInterval(id); }, []);
Closures capture the value at the time the effect ran. Omit a dep and its value freezes.

Pitfall 3 — object/array deps always "changed"

// ❌ options is a new object on every render const options = { includeDeleted: false }; // recreated each render! useEffect(() => { fetch(userId, options); }, [userId, options]); // infinite! // ✅ Define stable reference outside the component const OPTIONS = { includeDeleted: false }; useEffect(() => { fetch(userId, OPTIONS); }, [userId]); // ✓
React compares deps with Object.is(). Two objects with same contents are still different references.

Pitfall 4 — side effects in render body

// ❌ Runs during render! function MyComp() { localStorage.setItem('k', val); return <div />; }
// ✅ After render function MyComp() { useEffect(() => { localStorage.setItem('k', val); }, [val]); return <div />; }
Render must be a pure function. React 18 Strict Mode double-invokes renders to surface this mistake.
